# Generated Analysis Outputs

This folder is generated by the analysis pipeline and is ignored by Git by
default. It is safe to regenerate from the active OMNeT++ result folders and
analysis scripts, but do not delete current outputs while preparing a package
or final report unless you intend to regenerate them immediately.

## Important Subfolders

- `datasets/`: baseline and extended telemetry datasets.
- `reports/`: dataset reports and feature-classification reports.
- `outcomes/`: scenario outcome summaries, headline summaries, and comparison
  reports.
- `network_impact/`: UDP/QoS, backup-cost, and mixed transport endpoint proxy
  summaries.
- `ml_audit/`: offline ML feasibility, feature-quality, timing, and feature
  importance outputs.
- `debug/`: pipeline integrity reports and focused trace/debug outputs used by
  diagnostics.
- `audit/`: implementation and public-readiness audit records.
- `final_evaluation/`: the self-contained final report folder with clean main,
  scenario, and diagnostic figures plus compact CSV tables.
- `current_experiment/`: compact scenario packages intended for review/sharing.
- `experiment_logs/`: generated wrapper execution logs; useful for local
  traceability but not usually needed in a public commit.
- `training/`: generated model-training support outputs.

## Cleanup Policy

Use dry-run cleanup first:

```bat
cmd /c run_analysis.bat clean-final-evaluation --dry-run
cmd /c run_analysis.bat clean-generated --dry-run
```

`clean-final-evaluation` targets only generated image files under
`final_evaluation` figure folders. It does not target CSV tables, reports,
datasets, outcomes, raw OMNeT++ results, runtime model CSVs, source files, or
packages.

## Public Sharing Guidance

For GitHub, source/config/docs should be committed; large generated outputs and
raw `.sca/.vec/.vci` files should usually stay ignored. If reviewers need a
compact snapshot, use:

```bat
cmd /c run_analysis.bat evaluate-results
cmd /c run_analysis.bat package-current-experiment --scenario regionalbackbone_failure_detection_cost_aware_transport_impact
```

The final report must keep conservative wording: UDP app[0] packet loss is
exact only for that monitored flow, TCP metrics are endpoint proxies, and
recovery time is a receiver-observed disruption/recovery proxy.
